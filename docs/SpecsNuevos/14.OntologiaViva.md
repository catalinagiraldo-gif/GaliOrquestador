# Spec 9 — OntologiaViva: Skills ≠ Agentes ≠ Reglas — Diferenciación total

> **Fuente**: `docs/SpecsNuevos/Cambios10jun.md` §2 (Spec 14) · §9 (Motion Language + Look & Feel global)  
> **Extiende**: `docs/SpecsNuevos/8.AuditoriaJun8.md` — C5 (rediseño cards skill), C6 (tab Config agente), C7 (autopilot centralizado)  
> **Fecha**: 2026-06-10  
> **Prioridad**: P1 — consolida la promesa conceptual de Gali (agentes, skills y reglas son cosas distintas)

---

## Diagnóstico

El usuario que llega a la sección de Agentes ve cuatro secciones de navegación de primer nivel:  
**Agentes · Skills · Reglas · Marketplace**. Cada una expone la arquitectura interna de Gali como si fuera una opción de usuario cotidiano, cuando el 90% de los dropshippers nunca necesitan distinguir entre una skill y una regla.

### Causas raíz detectadas (referencia: CR1 y CR4 del plan 10 Jun)

**CR1 — Arquitectura expuesta como UI**  
"Agentes", "Skills", "Reglas", "Marketplace" como ítems de nav de primer nivel contradicen la promesa de que Gali opera "de fondo".

**Problema concreto 1 — Skills con propiedades de Agentes**  
La interfaz `SkillRule` en `gali-skill-builder-v2.component.ts` define campos `trigger`, `condition` y `action`. Estos campos pertenecen conceptualmente a los **Agentes** (entidades que deciden cuándo actuar), no a las **Skills** (capacidades que el agente invoca). Aunque el template actual de `gali-skill-builder-v2.component.html` no los renderiza directamente, la interfaz expone esta confusión semántica y el mock `skills-page.component.ts` los incluye en cada SkillRule.

**Problema concreto 2 — El diagrama "Gali orquesta" en `skills-page.component.html`**  
Existe un grid de agentes con sus skills. Está bien estructurado, pero carece de la jerarquía visual que comunique: Gali (orquestador) → Agentes (ejecutores) → usan Skills → gobernados por Reglas. La leyenda actual solo dice "Skill = capacidad reutilizable · Regla = restricción". Falta la capa intermedia (el Agente) como protagonista.

**Problema concreto 3 — Autopilot en múltiples lugares**  
El `gali-workspace-mode-bar.component.html` tenía un toggle de Autopilot global (ya migrado a un link "Configurar autopilot →" en Spec 8 C7). Sin embargo, el **modo-bar** sigue siendo el punto de entrada al concepto de autopilot, no la ficha del agente. El usuario que quiere activar autopilot de Roax específicamente tiene que saber que debe ir a Agentes → seleccionar Roax → Tab Configuración. Ese camino no está señalizado.

**Problema concreto 4 — Marketplace mezclado con "Mis Skills"**  
En `skills-page.component.html`, el tab "Marketplace →" es un link a `/gali-v5/marketplace`, separado visualmente pero sin jerarquía propia. La pestaña comunidad (`.skills-main-tab--link`) usa estilos distintos a las demás tabs — no está a la misma altura conceptual. El usuario no diferencia si el Marketplace es una sección o una pestaña.

**Problema concreto 5 — Tipografía hardcodeada en comunidad**  
`skills-comunidad-page.component.scss` línea 808 usa `font-family: 'IBM Plex Sans', monospace` en lugar de la variable `$font-menu` del DS. Viola la regla ALS-4 extendida a tokens tipográficos.

**Problema concreto 6 — "Forks" en inglés**  
El campo `forks` y la etiqueta "Fork de Chatea Pro" en `skills-comunidad-page.component.ts` usan jerga de GitHub que el dropshipper promedio no reconoce. La visión de Gali usa lenguaje colombiano.

## Definiciones vinculantes (usar en TODO el prototipo a partir de este spec)

| Concepto | Qué es | Analogía colombiana | Dónde vive en la UI |
|---|---|---|---|
| **Skill** | Capacidad que un agente puede usar. NO tiene trigger propio. Tiene historial auditable. | "La destreza del mecánico" | `/gali-v5/skills` |
| **Agente** | Ejecutor especializado con objetivo, reglas propias y autonomía. Usa Skills. Tiene trigger/condición/acción. | "El mecánico especializado" | `/gali-v5/agentes` |
| **Regla** | Instrucción IF-THEN que define cuándo actúa un agente. En lenguaje natural. | "Las instrucciones del jefe al mecánico" | Tab "Configuración" del agente · `/gali-v5/reglas` |
| **Gali** | El orquestador que coordina agentes, interpreta el negocio y reporta al dropshipper. | "El gerente general" | Hub · Panel derecho · Header |

prefillRuleHelp(ruleId: string): void {
  const agent = this.selectedAgent();
  const rule = agent.activeRules.find(r => r.id === ruleId);
  if (!rule) return;
  const improved = `Si ${rule.descripcion.toLowerCase().replace(/\.$/, '')} → notificar con resumen de impacto (afectados, monto estimado) y esperar aprobación antes de ejecutar.`;
  const updated: AgentRule = { ...rule, descripcion: improved };
  const updatedAgent = {
    ...agent,
    activeRules: agent.activeRules.map(r => r.id === ruleId ? updated : r),
  };
  this.agentsList.update(list => list.map(a => a.id === agent.id ? updatedAgent : a));
  this.selectedAgent.set(updatedAgent);
  this.ws.showToast({
    id: `rule-improved-${ruleId}`,
    agente: 'Gali',
    message: 'Regla mejorada con contexto de impacto y aprobación',
    tipo: 'ok',
  });
}

### C5 — Onboarding de primera visita en Agentes: jerarquía en 20 segundos

**Archivos**: `agentes-page.component.ts`, `agentes-page.component.html`, `agentes-page.component.scss`

Al llegar por primera vez a `/gali-v5/agentes`, aparece un panel modal pequeño con la jerarquía **Gali → Agente → Skill → Regla** que se puede cerrar manualmente o desaparece automáticamente en 20 segundos.

**`agentes-page.component.ts`** — añadir señal y lógica en `ngOnInit`:

```typescript
import { Component, computed, inject, signal, OnInit, OnDestroy } from '@angular/core';

// Añadir a la clase AgentesPageComponent:
readonly showOntologyOnboarding = signal(false);
private ontologyTimer: ReturnType<typeof setTimeout> | null = null;

ngOnInit(): void {
  const seen = localStorage.getItem('gali_ontology_seen');
  if (!seen) {
    this.showOntologyOnboarding.set(true);
    this.ontologyTimer = setTimeout(() => {
      this.dismissOntologyOnboarding();
    }, 20000);
  }
}

ngOnDestroy(): void {
  if (this.ontologyTimer) clearTimeout(this.ontologyTimer);
}

dismissOntologyOnboarding(): void {
  this.showOntologyOnboarding.set(false);
  localStorage.setItem('gali_ontology_seen', 'true');
  if (this.ontologyTimer) {
    clearTimeout(this.ontologyTimer);
    this.ontologyTimer = null;
  }
}
```
