## Problema original

Ninguna pantalla del prototipo muestra el valor total que Gali ha entregado. El dropshipper no puede responder la pregunta más básica: *"¿Para qué sirve Gali?"* con números reales.

La propuesta de valor dice: "Gali ejecutó 47 acciones esta semana · ahorró $420.000 en novedades evitadas · escaló 2 campañas con ROAS >2.5x". Ese enunciado no existe en ningún lugar del producto.

**Consecuencia**: el usuario no puede justificar emocionalmente ni racionalmente por qué Gali es indispensable. Sin ese número, Gali es una promesa sin evidencia — y las promesas sin evidencia no crean adherencia al producto.

**Referencia directa**: la misma brecha fue detectada en Spec 6 C5 (`docs/SpecsNuevos/6.Finance.md`), que ya define el bloque "Lo que Gali hizo por ti este mes" en la wallet con 4 stats hardcodeados. Este spec lo generaliza: crea el modelo de datos real, el widget reutilizable, y la vista dedicada de historial.

#### gali-impact-widget.component.ts

```typescript
import { Component, OnInit, Input, signal, computed, inject } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterLink } from '@angular/router';
import { GaliImpactSummary, GaliAgenteId } from '../../models/gali-impact.model';
import WALLET from '../../../../../../../mocks/gali-v5/wallet-transactions.json';

@Component({
  selector: 'app-gali-impact-widget',
  standalone: true,
  imports: [CommonModule, RouterLink],
  templateUrl: './gali-impact-widget.component.html',
  styleUrls: ['./gali-impact-widget.component.scss'],
})
export class GaliImpactWidgetComponent implements OnInit {
  // 'hub' | 'agent' | 'wallet' — controla qué stats mostrar y el tamaño
  @Input() mode: 'hub' | 'agent' | 'wallet' = 'hub';
  // Filtra acciones por agente cuando mode === 'agent'
  @Input() agenteId?: GaliAgenteId;

  private ledger = (WALLET as any).financialSummary.galiImpactLedger ?? [];
  private summaries: GaliImpactSummary[] =
    (WALLET as any).financialSummary.galiImpactSummaries ?? [];
  private roi = (WALLET as any).financialSummary.galiRoi ?? {};

  readonly semanaActual = signal<GaliImpactSummary | null>(null);
  readonly semanaAnterior = signal<GaliImpactSummary | null>(null);
  readonly acumuladoMes = signal({ pesos: 0, acciones: 0 });
  readonly tendenciaPct = signal(0);
  readonly statsAgente = signal({ acciones: 0, pesos: 0, novedades: 0 });

  ngOnInit(): void {
    const sorted = [...this.summaries].sort(
      (a, b) => b.semana_iso - a.semana_iso
    );
    this.semanaActual.set(sorted[0] ?? null);
    this.semanaAnterior.set(sorted[1] ?? null);

    // Acumulado del mes desde ledger
    const mesActual = sorted[0]?.mes ?? 5;
    const accionesDelMes = this.ledger.filter(
      (a: any) => a.mes === mesActual && !a.requirio_aprobacion
    );
    const pesosAcumulados = accionesDelMes.reduce(
      (s: number, a: any) => s + (a.impacto_pesos ?? 0), 0
    );
    this.acumuladoMes.set({
      pesos: pesosAcumulados,
      acciones: accionesDelMes.length,
    });

    // Tendencia semana vs semana anterior
    const actual = sorted[0]?.pesos_ahorrados ?? 0;
    const anterior = sorted[1]?.pesos_ahorrados ?? 1;
    const pct = anterior > 0 ? Math.round(((actual - anterior) / anterior) * 100) : 0;
    this.tendenciaPct.set(pct);

    // Stats filtradas por agente (para mode === 'agent')
    if (this.agenteId) {
      const accionesAgente = this.ledger.filter(
        (a: any) => a.agente === this.agenteId && a.mes === mesActual
      );
      this.statsAgente.set({
        acciones: accionesAgente.length,
        pesos: accionesAgente.reduce(
          (s: number, a: any) => s + (a.impacto_pesos ?? 0), 0
        ),
        novedades: accionesAgente.filter(
          (a: any) =>
            a.tipo_accion === 'evito_novedad' ||
            a.tipo_accion === 'cambio_operadora' ||
            a.tipo_accion === 'optimizo_ruta_logistica'
        ).length,
      });
    }
  }

  formatPesos(val: number): string {
    if (val >= 1_000_000) return `$${(val / 1_000_000).toFixed(1)}M`;
    if (val >= 1_000) return `$${Math.round(val / 1_000)}k`;
    return `$${val}`;
  }

  get tendenciaLabel(): string {
    const p = this.tendenciaPct();
    if (p > 0) return `↑${p}% vs semana pasada`;
    if (p < 0) return `↓${Math.abs(p)}% vs semana pasada`;
    return 'igual que la semana pasada';
  }

  get tendenciaClass(): string {
    const p = this.tendenciaPct();
    return p > 0 ? 'up' : p < 0 ? 'down' : 'neutral';
  }

  get roiMultiplier(): string {
    return this.roi.roi_multiplicador ?? '—';
  }
}
```

import { Component, OnInit, signal, computed } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterLink } from '@angular/router';
import { GaliImpactWidgetComponent } from '../components/gali-impact-widget/gali-impact-widget.component';
import { GaliImpactAction, GaliImpactSummary, GaliImpactMilestone } from '../models/gali-impact.model';
import WALLET from '../../../../../../mocks/gali-v5/wallet-transactions.json';

type FiltroAgente = 'todos' | 'Roax' | 'Vigilante' | 'Kronos' | 'Brujula' | 'Gali';

@Component({
  selector: 'app-gali-impacto-page',
  standalone: true,
  imports: [CommonModule, RouterLink, GaliImpactWidgetComponent],
  templateUrl: './gali-impacto-page.component.html',
  styleUrls: ['./gali-impacto-page.component.scss'],
})
export class GaliImpactoPageComponent implements OnInit {
  readonly ledger: GaliImpactAction[] =
    (WALLET as any).financialSummary.galiImpactLedger ?? [];
  readonly summaries: GaliImpactSummary[] =
    (WALLET as any).financialSummary.galiImpactSummaries ?? [];
  readonly milestones: GaliImpactMilestone[] =
    (WALLET as any).financialSummary.galiImpactMilestones ?? [];

  readonly filtroAgente = signal<FiltroAgente>('todos');
  readonly accionesVisibles = computed(() => {
    const f = this.filtroAgente();
    return f === 'todos'
      ? this.ledger
      : this.ledger.filter(a => a.agente === f);
  });

  readonly acumuladoTotal = computed(() => ({
    pesos: this.ledger.reduce((s, a) => s + (a.impacto_pesos ?? 0), 0),
    acciones: this.ledger.length,
    autonomas: this.ledger.filter(a => !a.requirio_aprobacion).length,
  }));

  readonly semanaActual = signal<GaliImpactSummary | null>(null);
  readonly semanaAnterior = signal<GaliImpactSummary | null>(null);

  // Para la barra de progreso del próximo milestone
  readonly proximoMilestone = computed(() =>
    this.milestones.find(m => !m.alcanzado) ?? null
  );
  readonly milestoneProgress = computed(() => {
    const m = this.proximoMilestone();
    if (!m) return 100;
    const actual = this.acumuladoTotal().pesos;
    return Math.min(100, Math.round((actual / m.umbral) * 100));
  });

  readonly agenteFiltros: FiltroAgente[] = ['todos', 'Roax', 'Vigilante', 'Kronos', 'Brujula', 'Gali'];

  ngOnInit(): void {
    const sorted = [...this.summaries].sort((a, b) => b.semana_iso - a.semana_iso);
    this.semanaActual.set(sorted[0] ?? null);
    this.semanaAnterior.set(sorted[1] ?? null);
  }

  setFiltro(f: FiltroAgente): void {
    this.filtroAgente.set(f);
  }

  formatPesos(val: number): string {
    if (val >= 1_000_000) return `$${(val / 1_000_000).toFixed(1)}M`;
    if (val >= 1_000) return `$${Math.round(val / 1_000)}k`;
    return `$${val}`;
  }

  formatTimestamp(ts: string): string {
    const d = new Date(ts);
    return d.toLocaleDateString('es-CO', {
      day: 'numeric',
      month: 'short',
      hour: '2-digit',
      minute: '2-digit',
    });
  }

  getAgenteColor(agente: string): string {
    const map: Record<string, string> = {
      Roax:      '#f49a3d',
      Vigilante: '#22c55e',
      Kronos:    '#60a5fa',
      Brujula:   '#a78bfa',
      Gali:      '#f49a3d',
    };
    return map[agente] ?? '#9ca3af';
  }

  getTipoIcon(tipo: string): string {
    const map: Record<string, string> = {
      escalo_campana:            '📈',
      evito_novedad:             '🛡',
      detecto_senal_predictiva:  '🔮',
      optimizo_ruta_logistica:   '🚚',
      alerto_stock:              '⚠️',
      genero_diagnostico:        '🧠',
      cambio_operadora:          '🔄',
      escalo_presupuesto:        '💰',
    };
    return map[tipo] ?? '✦';
  }
}

### C4 — Cálculos desde mock: métricas derivadas

Los cálculos de impacto no se hardcodean en los componentes — se derivan del `galiImpactLedger` usando fórmulas documentadas aquí.

**Fórmulas autoritativas** (aplicar en `ngOnInit` o `computed`):

```typescript
// Pesos ahorrados totales = suma de impacto_pesos del ledger
pesos_ahorrados_total = ledger.reduce((s, a) => s + a.impacto_pesos, 0);

// Novedades evitadas = acciones de tipo evito_novedad | cambio_operadora | optimizo_ruta_logistica
novedades_evitadas = ledger.filter(
  a => ['evito_novedad', 'cambio_operadora', 'optimizo_ruta_logistica'].includes(a.tipo_accion)
).length;

// Campañas escaladas = acciones escalo_campana | escalo_presupuesto
campanas_escaladas = ledger.filter(
  a => ['escalo_campana', 'escalo_presupuesto'].includes(a.tipo_accion)
).length;

// Señales detectadas = acciones detecto_senal_predictiva
senales_detectadas = ledger.filter(a => a.tipo_accion === 'detecto_senal_predictiva').length;

// Horas libradas = acciones_autonomas × (20/60) — 20 min por acción
horas_libradas = ledger.filter(a => !a.requirio_aprobacion).length * (20 / 60);

// % autonomía = acciones_autonomas / acciones_totales × 100
pct_autonomia = (acciones_autonomas / acciones_totales) * 100;
```

**Regla**: Nunca recalcular estos valores en múltiples componentes. Si el widget y la página de impacto necesitan los mismos datos, extraer a un `GaliImpactService` (signal-based) que ambos inyectan.

## Decisiones de diseño

**Por qué el widget tiene 3 modos**: El impacto de Gali necesita aparecer en múltiples contextos con diferentes densidades. El modo `hub` es colapsable y discreto (Momento 3). El modo `agent` es una línea inline (no compite con el resto de la ficha). El modo `wallet` es el más expandido (el contexto financiero lo justifica). Un solo componente con `@Input() mode` evita duplicar lógica y mantiene la fuente única de datos.

**Por qué el ledger vive en wallet-transactions.json y no en un archivo propio**: Spec 6 C5 ya estableció `financialSummary.galiRoi` como el punto de partida. Extender ese objeto mantiene la coherencia con lo ya implementado. En una implementación real, esto migraría a su propio endpoint — en el prototipo, la colocalización en wallet-transactions.json es la forma más rápida de tener la vista financiera y la de impacto leyendo del mismo mock.

**Por qué `pesos_ahorrados` se acumula desde el ledger y no desde `galiRoi.pesos_ahorrados`**: El ledger es la fuente atomística — cada acción tiene su impacto individual. El `galiRoi` es un snapshot mensual para el widget rápido de la wallet. Si hay discrepancia entre los dos, el ledger prevalece (los datos atómicos son más confiables que un agregado hardcodeado).

**Motion language (§9.0)**: Los contadores de impacto ($247k, 47 acciones) usan `gali-count-in` para comunicar que son valores calculados, no etiquetas estáticas. El stagger de 80ms entre stat cards crea la sensación de "recibir información", no de "ver una pantalla cargada". Esto es coherente con la promesa narrativa del spec: Gali reporta, no acumula.
