> **Fuente**: `docs/SpecsNuevos/Cambios10jun.md` §2 (Spec 11) · §9.0 (Motion Language Global) · `docs/SpecsNuevos/8.AuditoriaJun8.md` (C8/C9 base)

> **Dependencia**: Spec 8 debe estar ejecutado antes de este spec (C8/C9 del Spec 8 son la base que este spec extiende y eleva)

## Diagnóstico del problema

### Estado actual post-Spec 8

Spec 8 resolvió los problemas funcionales básicos de la pantalla de agentes:
- `launchAgent()` ya persiste el agente creado en el array `agentsList` (signal)
- El wizard ya tiene 6 pasos con estructura de `CustomAgentDraft`
- La ficha del agente ya tiene tabs `actividad | skills | config`
- La tab "Configuración" ya tiene autopilot toggle, umbral, skills y reglas

Sin embargo, el componente sigue siendo **decorativo en lo más importante**: el usuario puede ver que los agentes "existen" pero no puede responder la pregunta más relevante: **"¿Qué está haciendo Roax ahora mismo y qué logró ayer?"**

### Causa raíz (CR original: "Agentes decorativos")

| Problema observable | Causa subyacente |
|---|---|
| La lista de agentes muestra estado como chip estático | No hay feed de actividad reciente en primer plano |
| El "Historial" en Tab Actividad es una lista plana sin filtros | No existe diferencia entre "ejecutó", "alertó", "esperó aprobación" |
| El wizard crea el agente pero la experiencia se cierra abruptamente | No hay animación "Gali configurando" que conecte el acto de crear con el de activar |
| "Autopilot ON" no es visible en el listado | El usuario no sabe cuáles agentes actúan solos sin mirarlo en el detalle |
| El wizard tiene 6 pasos pero Paso 3 (Regla) y Paso 5 (Revisión) no tienen el nivel visual necesario | La regla is-then no es visual; el resumen final es texto plano |
| Crear un agente se siente como rellenar un formulario | No hay chips de sugerencia de rol, no hay selector visual en revisión |

### Lo que Spec 11 resuelve

Este spec **no reimplementa** lo que Spec 8 construyó — lo eleva en tres dimensiones:

1. **Visibilidad de actividad**: cada agente en la lista muestra su estado en tiempo real y su última acción con delta de impacto
2. **Activity Feed como git log de negocio**: la tab "Historial" se convierte en un registro filtrável de decisiones con antes/después
3. **Wizard con polish de clase mundial**: 5 pasos rediseñados con regla if-then visual, revisión final tipo "resumen de CEO", y animación de activación

La función `launchAgent()` ya persiste el agente (Spec 8 C8). Spec 11 añade:

```typescript
launchAgent(): void {
  const d = this.createAgentDraft();
  const agentId = `custom-${Date.now()}`;

  const newAgent: Agent = {
    id: agentId,
    name: d.nombre || 'Mi Agente',
    role: d.rol || 'Personalizado',
    color: '#a855f7',
    status: 'esperando',        // estado inicial visible en card
    lastAction: 'Configurando…', // texto que agent-card-alive muestra como 'configurando'

    recentHistory: [],
    ctas: [{ label: 'Configurar acciones', icon: 'pi-cog', variant: 'primary' }],
    activeRules: d.reglas.trim()
      ? [{ id: `rule-${agentId}`, titulo: 'Regla inicial', descripcion: d.reglas.trim(), activa: true }]
      : [],

  this.agentsList.update(list => [...list, newAgent]);
  this.selectedAgent.set(newAgent);
  this.createAgentDone.set(true);

  this.ws.showToast({
    id: `agent-created-${agentId}`,
    agente: 'Gali',
    message: `Gali está configurando ${newAgent.name}...`,
    tipo: 'ok',
  });

  setTimeout(() => {
    const activated: Agent = {
      ...newAgent,
      status: 'activo',
      lastAction: 'Activo y listo para operar',
      lastActionTime: 'ahora',
    };
    this.agentsList.update(list => list.map(a => a.id === agentId ? activated : a));
    if (this.selectedAgent().id === agentId) {
      this.selectedAgent.set(activated);
    }
    this.ws.showToast({
      id: `agent-activated-${agentId}`,
      agente: 'Gali',
      message: `${newAgent.name} activado y listo`,
      tipo: 'ok',
    });
```

**Actualizaciones en el TS para los campos nuevos:**

```typescript
setDraftReglaCondicion(v: string): void {
  this.createAgentDraft.update(d => ({ ...d, reglaCondicion: v }));
}

setDraftReglaAccion(v: string): void {
  this.createAgentDraft.update(d => ({ ...d, reglaAccion: v }));
}

prefillReglaExample(): void {
  this.createAgentDraft.update(d => ({
    ...d,
    reglaCondicion: 'el CTR cae por debajo del 1% durante 48 horas',
    reglaAccion: 'pausar el adset y notificarme con un resumen del impacto',
  }));
}

// Getter para combinar condición + acción en el resumen del Paso 5
get reglaResumen(): string {
  const d = this.createAgentDraft();
  if (!d.reglaCondicion && !d.reglaAccion) return '';
  return `Si ${d.reglaCondicion} → ${d.reglaAccion}`;
}

// Actualizar nextCreateStep para el nuevo límite de 5
nextCreateStep(): void {
  this.createAgentDraft.update(d => ({
    ...d,
    step: Math.min(5, d.step + 1) as CustomAgentDraft['step'],
  }));
}

prevCreateStep(): void {
  this.createAgentDraft.update(d => ({
    ...d,
    step: Math.max(1, d.step - 1) as CustomAgentDraft['step'],
  }));
}
```
