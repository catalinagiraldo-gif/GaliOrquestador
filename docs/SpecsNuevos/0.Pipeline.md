## Reglas de ejecución

### Antes de cada spec

1. Leer el **archivo del spec completo** (problema, tareas, CA, OUT, lo que NO cambia, orden de implementación).
2. Spec **vacío o incompleto** → **detenerse** y reportar qué falta. **No improvisar alcance.**
3. Leer **skills** de §Skills (`SKILL.md` completo).
4. Antes de **UI**: DS Registry según el spec + `CLAUDE.md`. Spec 2: **`ds-registry/components/dropi-sidebar.json`** obligatorio.

### Durante el spec

- Respeta **"Lo que NO cambia"** y alcance OUT.
- **"Implementado" / "Hecho" / "Cierre" / ✅** → verificar CA; si cumple → **saltado/hecho**, siguiente spec **sin rehacer**.
- Diff **mínimo**; no items del §Backlog salvo que el spec activo los cite.

### Cierre de spec (gate N → N+1)

- [ ] Todos los **Criterios de aceptación** del spec en ✅ (o N/A documentado).
- [ ] **`ng build`** sin errores en archivos tocados.
- [ ] Sin violar **"Lo que NO cambia"**.
