## Problema

La auditoría DesADI identificó 14 inconsistencias de datos entre vistas del prototipo Gali V5. El mismo campo aparece con valores distintos en `home.component.ts` según el nodo del `nodeDetailMap` que se muestre. No existe ningún mock con autoridad sobre las métricas financieras por proyecto — todos los valores están hardcodeados en el TS con contradicciones internas.

**Inconsistencias críticas resueltas**:

| Campo | Antes (múltiples valores) | Después (canónico único) |
|---|---|---|
| ROAS real Collar GPS | `2.9x` (agentes, proyectos, marketing, rendimiento), `1.93x` (reportes) | **1.93x** — el real (Dropi) siempre gana |
| ROAS Meta declarado | `5.0x` (proyectosPL), `3.1x` (campanasMedir, rendimiento), `2.9x` (reportes) | **2.9x** — par canónico del nodo reportes |
| Margen neto | `34%` (marketing, rendimiento, reportes), `22%` (proyectosPL) | **22%** — valor real del P&L |
| Pauta escala propuesta | `$85k/día` (proyecciones financiero), `$86k/día` (insight, confirmScale) | **$86k/día** — valor de la acción aprobable |

### C1 — Propagar ROAS canónico vía helper/servicio

**Problema raíz**: `2.9x` (ROAS Meta declarado de Collar GPS) aparece en 15+ lugares como si fuera el ROAS real del negocio. El real es `1.93x`; el `2.9x` solo es válido cuando se etiqueta explícitamente como "ROAS Meta declarado".

**Solución**: Crear un helper o constante central que exporte los valores canónicos y reemplazar todos los string hardcodeados.

**Opción recomendada** — constante en `projects.json` ya existe (`kpis_por_proyecto[0].roas_real: 1.93`). Crear un helper TS en `src/app/pages/gali-v5/services/gali-data.helper.ts`:

```ts
import PROJECTS from '../../../../../../mocks/gali-v5/projects.json';
import KPIS from '../../../../../../mocks/gali-v5/kpis-global.json';
const GPS = PROJECTS.find(p => p.id === 'collar-gps-2026')!;
export const ROAS_REAL   = GPS.roas_real;          // 1.93
export const ROAS_META   = GPS.roas_meta;          // 2.9
export const PAUTA_DIARIA = GPS.pauta_diaria;      // 66000
export const ROAS_GLOBAL  = KPIS.roas_efectivo_global.valor; // 1.91
```
