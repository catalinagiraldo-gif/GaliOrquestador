## 1. Diagnóstico — Por qué no existe esta vista hoy

### 1.1 Ausencia total del enfoque predictivo

El botón "Señales" existe en el Hub pero no tiene destino. Lo que hay en Decisiones pendientes son **alertas puramente reactivas** (novedad de Coordinadora, cambio de transportadora). No hay ninguna pantalla que muestre lo que Gali ve *antes* de que el problema ocurra.

Cita directa de CorrecionesGali5.md:
> "La mitad del valor diferencial de Gali — el carácter predictivo — no se prototipa. No hay ningún ejemplo de lo que se definió como señal: 'tendencia de mercado que se abre en tu segmento', 'oportunidad de escalar antes de que llegue la competencia'."

### 1.2 Data del macromundo de Dropi invisible

El diferenciador más fuerte de Gali vs. cualquier IA genérica es que cruza datos globales de toda la red Dropi con el micromundo del usuario. Hoy ese cruce es invisible: las recomendaciones de ADA Spy aparecen sin decir de dónde vienen.

El patrón correcto (GaliAjustesJun3.md Bloque 2):
```
Macromundo: "Coordinadora 15% novedad en Bogotá esta semana"
Micromundo: "Este usuario tiene 23 pedidos activos hacia Bogotá"
Output:     "Cambié 23 pedidos a Servientrega. Ahorré ~4 novedades."
```

Sin el primer elemento visible, el usuario no entiende por qué Gali hace lo que hace.

### 1.3 Loop cerrado sin ruta propia

DesADIGali5.md §8: La promesa "Vista Señales diferenciada" está en estado **Parcial** — existen cards `gali-signal-card-v2` con tipos (crítica, oportunidad, completada) en el Hub, pero no hay ruta propia, y chat y señales compiten en el mismo layout.

El loop de acción (AlcancesIADropi.md VI) requiere una superficie dedicada:
```
Señal detectada
    → Acción tomada (con contexto de por qué)
    → Resultado medible (before / after)
    → Siguiente paso propuesto (crear skill, ajustar regla)
```

Sin ruta propia, este loop no puede ocurrir sin que el usuario abandone el Hub.

## 2. Distinción Señales vs. Alertas

Esta distinción es la columna vertebral del componente. Son dos conceptos fundamentalmente distintos y deben verse distintos:

| Dimensión | SEÑAL | ALERTA |
|---|---|---|
| **Naturaleza** | Predictiva — algo que *va a pasar* o *puede aprovecharse* | Reactiva — algo que *está pasando* y requiere acción |
| **Urgencia** | Ventana de 7-14 días | Requiere acción hoy o esta semana |
| **Origen** | Data global de la red Dropi cruzada con el micromundo del usuario | Estado actual de pedidos, campañas, conexiones |
| **Agente** | ADA Spy, Roax (cuando detectan tendencias antes de actuar) | Vigilante, Chatea Pro, Kronos (cuando detectan problemas activos) |
| **Tono** | Estratégico, orientado a oportunidad | Urgente, orientado a solución |
| **CTA** | "Aprovechar", "Explorar", "Aprobar escala" | "Resolver ahora", "Cambiar transportadora", "Conectar" |
| **Fondo visual** | `var(--surface-50)` — blanco muy sutil, tono neutro-cálido | `var(--orange-50)` o `var(--red-50)` según criticidad |

### 5.2 `senales-page.component.ts`

```typescript
import { Component, signal, computed } from '@angular/core';
import { CommonModule } from '@angular/common';
import { Router } from '@angular/router';
import { TooltipModule } from 'primeng/tooltip';
import { MOCK_SENALES, MOCK_ALERTAS } from '../../../../../mocks/gali-v5/senales.mock';

type FilterTab = 'todas' | 'senales' | 'alertas' | 'completadas';

@Component({
  selector: 'app-senales-page',
  standalone: true,
  imports: [CommonModule, TooltipModule],
  templateUrl: './senales-page.component.html',
  styleUrls: ['./senales-page.component.scss'],
})
export class SenalesPageComponent {
  readonly senales = MOCK_SENALES;
  readonly alertas = MOCK_ALERTAS;

  activeFilter = signal<FilterTab>('todas');

  senalesVisibles = computed(() =>
    this.activeFilter() === 'alertas' ? [] :
    this.activeFilter() === 'completadas'
      ? this.senales.filter(s => s.tipo === 'completed')
      : this.senales.filter(s => s.tipo !== 'completed')
  );

  alertasVisibles = computed(() =>
    this.activeFilter() === 'senales' ? [] :
    this.activeFilter() === 'completadas' ? [] :
    this.alertas
  );

  setFilter(f: FilterTab) { this.activeFilter.set(f); }

  // Fase 1: log o toast. Fase 2 (post Spec 3): abrir ConfirmActionModalComponent
  handleAlertaCta(alerta: any) {
    console.log('alerta CTA', alerta.id);
  }

  handleSenalCta(senal: any) {
    // TODO Sprint 2: conectar acción real
    console.log('señal CTA', senal.id);
  }
}
```
