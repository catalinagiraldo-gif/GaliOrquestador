## Diagnóstico del problema

Gali promete democratizar la consultoría de alto nivel para dropshippers colombianos. Esa promesa falla en el momento en que el usuario ve "PIL", "ROAS real vs declarado", "diagnóstico cruzado", "huella vigilante", "CPA" o "LTV" sin entender qué significan esos términos en el contexto concreto de su operación.

### Estado actual de la explicabilidad

| Término | Dónde aparece | Estado actual |
|---|---|---|
| ROAS | Dashboard Financiero, Hub, proyectos | `<abbr title="...">` o `[galiInsight]` con texto analítico — no educativo |
| PIL | Dashboard Financiero — sección `fin-glosario` estática | Bloque de glosario al fondo de la página, no inline |
| P&L | Dashboard Financiero | `<abbr title="P&L = Pérdidas y Ganancias...">` hardcodeado |
| CPA | Dashboard Financiero — sección `fin-glosario` | Estático, no contextual |
| LTV | Dashboard Financiero — sección `fin-glosario` | Estático, no contextual |
| Huella Vigilante | `orders-page` | `[galiInsight]` con texto analítico ("Rastro de acciones que tomó Vigilante...") |
| Novedad | `torre-logistica` | `[galiInsight]` con umbral operativo — no educativo |
| Diagnóstico cruzado | Hub, Señales | Sin tooltip |
| "Ferrín completado" | `micromundo-page` | Ya corregido → "Perfil completado" (no requiere cambio) |

### Las dos directivas que ya existen — y por qué no son suficientes

1. **`GaliInsightDirective`** (`src/app/pages/gali-v5/directives/gali-insight.directive.ts`): tooltip flotante de fondo oscuro (`#1a1a22`), etiqueta "Gali Insight". Diseñado para **análisis contextual** de métricas ("ROAS sobre objetivo — Gali considera escalar"). No es educativo: asume que el usuario ya sabe qué es ROAS y le da la interpretación de Gali. **No sirve para enseñar.**

2. **`GaliTermTooltipComponent`** (`src/app/pages/gali-v5/components/shared/gali-term-tooltip.component.ts`): ⓘ con PrimeNG `pTooltip`. Muestra texto plano en tooltip de PrimeNG. Sin dos niveles, sin adaptatividad, sin localStorage tracking.

### Lo que falta: `GaliGlosarioDirective`

Una directiva nueva, diferente en diseño y comportamiento:

| Característica | `galiInsight` | `gali-term-tooltip` | **`galiGlosario` (nuevo)** |
|---|---|---|---|
| Propósito | Análisis de datos | Definición breve | **Educación contextual con ejemplo** |
| Visual | Dark, flotante | PrimeNG tooltip | Light, anclado, expandible |
| Niveles | 1 | 1 | **2 (simple + "Ver más" con ejemplo)** |
| Adaptatividad | No | No | **Sí — desaparece tras 3 vistas** |
| Icono ⓘ inline | No | Componente separado | **Inline, parte de la directiva** |

### Los 5 problemas sistémicos que este spec ataca

**PS-1 — Glosario estático al fondo de página**: El usuario que no llega al scroll inferior nunca ve las definiciones de PIL, ROAS, CPA. El contexto educativo debe estar **donde el término aparece**, no donde el diseñador lo puso.

**PS-2 — Tooltip analítico ≠ tooltip educativo**: `galiInsight` da interpretación ("ROAS sobre objetivo — escalar"). Eso supone que el usuario ya sabe qué es ROAS. El glosario debe dar la definición primero, la interpretación después.

**PS-3 — Sin adaptatividad = ruido permanente**: Si el ⓘ nunca desaparece, el usuario experto lo ve como ruido visual. El sistema debe recordar cuántas veces vio cada término y silenciarse solo.

**PS-4 — Onboarding con preguntas que Dropi ya conoce**: Los steps de `pedidosPerWeek` y `tiempoEnDropi` (si aún existen en la plantilla HTML) preguntan datos que `kpis-global.json` ya tiene. El resultado: el usuario siente que Gali no lo conoce, exactamente lo contrario de la promesa.

**PS-5 — Contexto vacío para usuarios nuevos**: Un usuario con 0 pedidos ve las mismas preguntas que uno con 500. El onboarding debe bifurcar con mensajes propios para cada segmento (ya parcialmente implementado — verificar y completar).

### C1 — Crear directiva `GaliGlosarioDirective`

**Archivo**: `src/app/pages/gali-v5/directives/gali-glosario.directive.ts` ← **NUEVO**

Esta directiva envuelve cualquier término con un ⓘ inline que abre un tooltip de dos niveles con tracking de vistas en localStorage.

```typescript
import {
  Directive, ElementRef, Input, OnInit, OnDestroy,
  Renderer2, HostListener
} from '@angular/core';

export interface GlosarioEntry {
  /** 1 línea — qué es el término */
  simple: string;
  /** Ejemplo práctico concreto para el dropshipper (opcional) */
  ejemplo?: string;
}

// Diccionario canónico. Cualquier componente que use la directiva con `galiGlosario="ROAS"`
// obtiene automáticamente la definición correcta sin hardcodear el texto.
export const GLOSARIO: Record<string, GlosarioEntry> = {
  ROAS: {
    simple: 'Por cada $1 que gastas en pauta, cuánto en ventas recuperas. ROAS 2x = $2 en ventas por $1 invertido.',
    ejemplo: 'Si pagas $100.000 en Meta Ads y vendes $193.000, tu ROAS es 1.93x. El mínimo para no perder depende de tu margen (generalmente >2x).',
  },
  PIL: {
    simple: 'Porcentaje de Inventario Liquidable: qué tan fácil se venden tus productos vs cuánto tienes en stock.',
    ejemplo: 'PIL alto (>70%) significa que casi todo lo que tienes se está vendiendo. PIL bajo (<30%) indica que tienes producto empantanado.',
  },
  'P&L': {
    simple: 'Pérdidas y Ganancias: cuánto dinero real te quedó después de restar todos los costos (producto, pauta, flete, novedades).',
    ejemplo: 'Si vendiste $5M pero pagaste $3M en pauta + flete + novedades + comisiones, tu P&L real es $2M — no los $5M que muestra Meta.',
  },
  CPA: {
    simple: 'Costo por Adquisición: cuánto te costó conseguir una venta, incluyendo pauta, flete y novedades.',
    ejemplo: 'Si gastas $200.000 en pauta y consigues 10 pedidos, tu CPA de pauta es $20.000. Pero si 15% son novedades, el CPA real sube a ~$25.000.',
  },
  LTV: {
    simple: 'Valor de vida del cliente: cuánto en total te comprará un cliente durante toda su relación contigo.',
    ejemplo: 'Si un cliente te compra en promedio 2 veces al año durante 1.5 años con ticket promedio de $80.000, su LTV es $240.000.',
  },
  'Diagnóstico cruzado': {
    simple: 'Gali compara todos tus proyectos entre sí para encontrar patrones de éxito y fracaso.',
    ejemplo: 'Gali detecta que los 3 proyectos que más venden tienen en común: pauta >$50k/día + ROAS >2.5x en la primera semana + margen >35%.',
  },
  'Huella Vigilante': {
    simple: 'El registro de cada decisión que tomó Vigilante en este pedido: qué detectó, qué hizo y cuándo.',
    ejemplo: 'Vigilante detectó que la dirección estaba en zona de alta novedad (Cali Norte) → cambió a Servientrega → pedido entregado sin problema.',
  },
  Novedad: {
    simple: 'Cualquier problema en la entrega de un paquete: cliente no contestó, dirección incorrecta o paquete devuelto.',
    ejemplo: 'Una tasa de novedad del 15% significa que 15 de cada 100 pedidos tienen algún problema. El promedio sano en Colombia es <8%.',
  },
};

const STORAGE_PREFIX = 'gali_glosario_seen_';
const MAX_VIEWS_BEFORE_HIDE = 3;

@Directive({
  selector: '[galiGlosario]',
  standalone: true,
})
export class GaliGlosarioDirective implements OnInit, OnDestroy {
  /** Clave del diccionario GLOSARIO o texto inline como objeto {simple, ejemplo?} */
  @Input('galiGlosario') term: string | GlosarioEntry = '';

  private icon: HTMLElement | null = null;
  private panel: HTMLElement | null = null;
  private expanded = false;

  constructor(
    private el: ElementRef<HTMLElement>,
    private renderer: Renderer2,
  ) {}

  ngOnInit(): void {
    const entry = this.resolveEntry();
    if (!entry) return;

    const termKey = typeof this.term === 'string' ? this.term : 'custom';
    const views = this.getViewCount(termKey);

    // No mostrar ⓘ si el usuario ya vio el término más de MAX_VIEWS veces
    if (views >= MAX_VIEWS_BEFORE_HIDE) return;

    // Wrap host en un span relativo para que el panel se ancle correctamente
    const host = this.el.nativeElement;
    this.renderer.setStyle(host, 'position', 'relative');
    this.renderer.setStyle(host, 'display', 'inline-flex');
    this.renderer.setStyle(host, 'align-items', 'center');
    this.renderer.setStyle(host, 'gap', '3px');

    // Crear ⓘ icon
    const icon = this.renderer.createElement('span') as HTMLElement;
    icon.className = 'gali-glosario-icon';
    icon.textContent = 'ⓘ';
    icon.setAttribute('aria-label', `Definición de ${termKey}`);
    icon.setAttribute('role', 'button');
    icon.setAttribute('tabindex', '0');
    icon.style.cssText = `
      font-size: 10px;
      color: #858ea6;
      cursor: help;
      flex-shrink: 0;
      line-height: 1;
      transition: color 0.12s ease;
      user-select: none;
    `;

    icon.addEventListener('mouseenter', () => {
      icon.style.color = '#f49a3d';
      this.showPanel(entry, termKey);
    });
    icon.addEventListener('mouseleave', () => {
      icon.style.color = '#858ea6';
    });
    icon.addEventListener('click', (e) => {
      e.stopPropagation();
      this.togglePanel(entry, termKey);
    });

    this.renderer.appendChild(host, icon);
    this.icon = icon;
  }

  ngOnDestroy(): void {
    this.destroyPanel();
  }

  @HostListener('document:click')
  onDocumentClick(): void {
    this.destroyPanel();
  }

  // ── Helpers ────────────────────────────────────────────────────────────────

  private resolveEntry(): GlosarioEntry | null {
    if (typeof this.term === 'string') {
      return GLOSARIO[this.term] ?? null;
    }
    return this.term;
  }

  private showPanel(entry: GlosarioEntry, termKey: string): void {
    if (this.panel) return;
    this.incrementViewCount(termKey);
    this.buildPanel(entry, termKey);
  }

  private togglePanel(entry: GlosarioEntry, termKey: string): void {
    if (this.panel) {
      this.destroyPanel();
    } else {
      this.showPanel(entry, termKey);
    }
  }

  private buildPanel(entry: GlosarioEntry, termKey: string): void {
    const host = this.el.nativeElement;
    const panel = this.renderer.createElement('div') as HTMLElement;
    panel.className = 'gali-glosario-panel';

    const viewsLeft = Math.max(0, MAX_VIEWS_BEFORE_HIDE - this.getViewCount(termKey));

    panel.innerHTML = `
      <div class="ggp__header">
        <span class="ggp__term">${typeof this.term === 'string' ? this.term : 'Término'}</span>
        ${viewsLeft === 1 ? `<span class="ggp__fade-hint">Última vez que te lo muestro</span>` : ''}
      </div>
      <p class="ggp__simple">${entry.simple}</p>
      ${entry.ejemplo ? `
        <button class="ggp__ver-mas" type="button">Ver ejemplo →</button>
        <p class="ggp__ejemplo" style="display:none">${entry.ejemplo}</p>
      ` : ''}
    `;

    panel.style.cssText = `
      position: absolute;
      top: calc(100% + 6px);
      left: 0;
      z-index: 1000;
      background: #ffffff;
      border: 1px solid #e8eaf0;
      border-left: 3px solid #f49a3d;
      border-radius: 8px;
      padding: 10px 12px;
      min-width: 240px;
      max-width: 300px;
      box-shadow: 0 4px 16px rgba(0,0,0,0.1);
      font-family: 'Inter', sans-serif;
      animation: gali-fade-up 0.15s ease-out forwards;
      pointer-events: auto;
    `;

    // Expandir ejemplo al click en "Ver más"
    const verMasBtn = panel.querySelector('.ggp__ver-mas') as HTMLElement | null;
    const ejemploEl = panel.querySelector('.ggp__ejemplo') as HTMLElement | null;
    if (verMasBtn && ejemploEl) {
      verMasBtn.addEventListener('click', (e) => {
        e.stopPropagation();
        const hidden = ejemploEl.style.display === 'none';
        ejemploEl.style.display = hidden ? 'block' : 'none';
        verMasBtn.textContent = hidden ? 'Ocultar ejemplo ↑' : 'Ver ejemplo →';
        this.expanded = hidden;
      });
    }

    // Evitar que click en el panel cierre el documento
    panel.addEventListener('click', (e) => e.stopPropagation());

    this.renderer.appendChild(host, panel);
    this.panel = panel;

    // Si no quedan vistas, ocultar el ⓘ tras cerrar el panel
    if (this.getViewCount(termKey) >= MAX_VIEWS_BEFORE_HIDE && this.icon) {
      setTimeout(() => {
        if (this.icon) {
          this.icon.style.transition = 'opacity 0.3s ease';
          this.icon.style.opacity = '0';
          setTimeout(() => this.icon?.remove(), 300);
          this.icon = null;
        }
      }, 2000);
    }
  }

  private destroyPanel(): void {
    if (this.panel) {
      this.panel.remove();
      this.panel = null;
      this.expanded = false;
    }
  }

  private getViewCount(termKey: string): number {
    return parseInt(localStorage.getItem(`${STORAGE_PREFIX}${termKey}`) ?? '0', 10);
  }

  private incrementViewCount(termKey: string): void {
    const current = this.getViewCount(termKey);
    localStorage.setItem(`${STORAGE_PREFIX}${termKey}`, String(current + 1));
  }
}
```

### C8 — Tooltips adaptativos: desaparece el ⓘ tras 3 vistas

Ya implementado dentro de `GaliGlosarioDirective` (C1). Mecánica:

1. Cada vez que el panel se muestra, `incrementViewCount(termKey)` guarda en `localStorage` el conteo bajo la clave `gali_glosario_seen_<termKey>`.
2. En `ngOnInit()`, si `getViewCount(termKey) >= MAX_VIEWS_BEFORE_HIDE (3)`, el ⓘ no se renderiza.
3. Cuando el panel se cierra y el conteo alcanza 3, el ⓘ hace fade-out (opacity 0 → 0 en 300ms) y se elimina del DOM.

**Verificar el comportamiento en browser**:
- Abrir Dashboard Financiero → hover sobre "PIL" → panel aparece (vista 1/3)
- Cerrar y reabrir → panel aparece (vista 2/3)
- A la tercera apertura: "Última vez que te lo muestro" aparece en el header del panel
- Al cerrar la tercera vez: el ⓘ hace fade y desaparece
- Reload de página: "PIL" ya no muestra el ⓘ (localStorage persiste)
- Para resetear en dev: `localStorage.removeItem('gali_glosario_seen_PIL')` en la consola del browser

**Reset para demo**: añadir función de utilidad en `micromundo-page.component.ts` (tab "Perfil") o en el onboarding para limpiar el estado del glosario si se desea resetear la experiencia:

```typescript
// micromundo-page.component.ts — función de utilidad para demo:
resetGlosario(): void {
```

## Consideraciones de diseño

### Por qué no reutilizar `galiInsight`

`galiInsight` es un tooltip **de análisis**: responde "¿qué significa este número en mi contexto?" con datos específicos ("ROAS sobre objetivo — considera escalar"). Requiere contexto dinámico del negocio.

`galiGlosario` es un tooltip **educativo**: responde "¿qué es este concepto?" con una definición fija más un ejemplo concreto. No necesita datos del negocio — sirve igual para un usuario con 0 pedidos que para uno con 500.

Los dos deben coexistir en la misma pantalla. Ejemplo en el Dashboard:
- **Label "ROAS real"**: tiene `galiGlosario="ROAS"` → enseña qué es ROAS
- **Valor "1.93×"**: tiene `[galiInsight]="'Gali detectó tendencia alcista...'"` → interpreta el número

### Visual: light vs dark

`galiInsight` usa fondo oscuro (`#1a1a22`) para comunicar "inteligencia de IA", como si Gali hablara.  
`galiGlosario` usa fondo blanco con borde naranja para comunicar "enciclopedia amable", como si Gali enseñara.  
Este contraste visual ayuda al usuario a distinguir cuándo Gali está interpretando vs educando.

### Accesibilidad

- El ⓘ tiene `role="button"` + `tabindex="0"` → navigable con teclado
- El panel se cierra con `document:click` → click fuera lo cierra
- `aria-label` en el ⓘ identifica el término para screen readers
