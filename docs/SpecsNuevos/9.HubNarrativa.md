## 1. Diagnóstico — Por qué el Hub sigue sin funcionar

### 1.1 Causa raíz: arquitectura expuesta como UI (CR1)

El Hub acumula **7 bloques de información en un viewport** compitiendo simultáneamente sin jerarquía de lectura clara: barra de intent · tabs de dashboard · banner de objetivo · banner de alerta de agente · ciclo de negocio (6 tarjetas) · decisiones pendientes · chat con sugerencias · Academy recomendado. El usuario no sabe qué atender primero.

**Heurística Nielsen #8** — Diseño estético y minimalista: es el problema más grave del prototipo actual.

### 1.2 El impacto de Gali es invisible (CR5)

La propuesta dice "Gali te ahorra tiempo y dinero". En ningún lugar del Hub existe el número "Gali ejecutó 47 acciones esta semana · ahorró $420.000 en novedades evitadas · escaló 2 campañas con ROAS >2.5x". Sin ese número, el usuario no puede justificar emocionalmente por qué Gali es indispensable.

### 1.3 Sin jerarquía de urgencia transversal (CR2)

No existe un sistema centralizado de alertas. Cada módulo genera su propia alerta con su propia prioridad visual. El resultado: 3-4 banners compitiendo en cada pantalla, ninguno claramente "el más importante". El usuario desarrolla ceguera de banners (*banner blindness*).

### 1.4 Gali como dashboard, no como Director

La pantalla nunca dice "Tu negocio va bien esta semana. Hay 1 decisión que necesito de ti y 3 cosas que ya resolví." — esa es la voz de un Director de E-commerce. El Hub actual es la voz de un tablero de métricas. La diferencia entre los dos determina si el producto es una herramienta o un socio.

## 2. Estructura propuesta — 3 Momentos secuenciales

El usuario lee el Hub de arriba a abajo como una historia de negocio. Cada momento responde a una pregunta única:

```
┌─────────────────────────────────────────────────────────────────┐
│  MOMENTO 1 — El resumen de Gali           (siempre visible)     │
│  "Hola [nombre]. Tu negocio va bien esta semana.               │
│   Hay 1 decisión que necesito de ti y 3 cosas que ya resolví." │
│  [Ver qué resolví →]   [Ver la decisión →]                      │
├─────────────────────────────────────────────────────────────────┤
│  MOMENTO 2 — La decisión pendiente        (máximo 1 a la vez)   │
│  Full-width. Un solo card con TODA la info necesaria.           │
│  Contexto · Riesgo · Opciones · Impacto estimado               │
│  [Opción A]   [Opción B]   [Decidir después →]                  │
├─────────────────────────────────────────────────────────────────┤
│  MOMENTO 3 — El estado del negocio        (colapsable, experto) │
│  Ciclo de negocio: Producto→Marketing→Pedidos→Logística→Fin    │
│  "Gali esta semana" (ROI acumulado)                             │
│  Academy pill                                                    │
└─────────────────────────────────────────────────────────────────┘
```

### Regla de Anti-Saturación (ALS-3 integrada en este spec)

- El Hub **nunca** muestra más de 1 decisión pendiente a la vez en pantalla
- Las otras decisiones están en cola: badge con contador en el header (`pendingDecisions - 1`)
- El Momento 3 (ciclo de negocio) solo aparece si el usuario lo activa — o en modo experto (`complexityLevel === 'expert'`)
- Si no hay decisiones pendientes y todo está OK: el Hub muestra un estado vacío positivo (Modo calma, ver C6)

### C1 — Crear componente `gali-hub-brief` (Momento 1)

```typescript
import { Component, computed, input, output } from '@angular/core';
import { CommonModule } from '@angular/common';

export interface HubBriefData {
  userName: string;
  pendingDecisions: number;
  completedActions: number;
  roasReal: number;
  roasObjective: number;
  alertasCriticas: number;
  ahorroSemanal: string;
  proyectosPrincipales: string[];
}

@Component({
  selector: 'gali-hub-brief',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './gali-hub-brief.component.html',
  styleUrls: ['./gali-hub-brief.component.scss']
})
export class GaliHubBriefComponent {
  data = input.required<HubBriefData>();

  onVerDecisiones = output<void>();
  onVerResuelto  = output<void>();

  readonly estadoNegocio = computed<'bien' | 'atencion' | 'critico'>(() => {
    const d = this.data();
    if (d.alertasCriticas > 0)       return 'critico';
    if (d.roasReal < d.roasObjective) return 'atencion';
    return 'bien';
  });

  readonly tituloBrief = computed<string>(() => {
    const d    = this.data();
    const hora = new Date().getHours();
    const saludo = hora < 12 ? 'Buenos días' : hora < 18 ? 'Buenas tardes' : 'Buenas noches';
    const nombre = d.userName.split(' ')[0];

    switch (this.estadoNegocio()) {
      case 'bien':
        return `${saludo}, ${nombre}. Tu negocio va bien esta semana.`;
      case 'atencion':
        return `${saludo}, ${nombre}. Tu negocio necesita atención.`;
      case 'critico':
        return `${saludo}, ${nombre}. Hay situaciones urgentes hoy.`;
    }
  });

  readonly subtextoBrief = computed<string>(() => {
    const d = this.data();
    const pendientes = d.pendingDecisions;
    const resueltas  = d.completedActions;

    if (pendientes === 0 && resueltas === 0) {
      return 'Todo está corriendo sin novedades. Gali está monitoreando tu operación.';
    }
    if (pendientes === 0) {
      return `Gali resolvió ${resueltas} ${resueltas === 1 ? 'cosa' : 'cosas'} automáticamente. No hay decisiones pendientes.`;
    }
    if (resueltas === 0) {
      return `Hay ${pendientes} ${pendientes === 1 ? 'decisión que necesita' : 'decisiones que necesitan'} tu aprobación.`;
    }
    return `Hay ${pendientes} ${pendientes === 1 ? 'decisión que necesita' : 'decisiones que necesitan'} tu aprobación y ${resueltas} ${resueltas === 1 ? 'cosa que ya resolví' : 'cosas que ya resolví'}.`;
  });

  readonly mostrarBotonResuelto = computed(() => this.data().completedActions > 0);
  readonly mostrarBotonDecisiones = computed(() => this.data().pendingDecisions > 0);
}
```

### C2 — Crear componente `gali-decision-theater` (Momento 2)

Una sola decisión full-width con todo el contexto necesario para decidir en 10 segundos. El "teatro" es que la decisión tiene el escenario completo: contexto, riesgo, opciones, impacto estimado.

**Archivo nuevo**: `src/app/pages/gali-v5/components/gali-decision-theater/gali-decision-theater.component.ts`

```typescript
import { Component, input, output, computed } from '@angular/core';
import { CommonModule } from '@angular/common';
import { GaliSignal } from '../../models/gali-signal.model'; // modelo existente del Spec 7

export type DecisionUrgency = 'critical' | 'high' | 'medium';

export interface DecisionOption {
  id: string;
  label: string;
  description: string;
  impactoEstimado: string;
  isPrimary: boolean;
}

export interface DecisionTheaterData {
  signalId: string;
  agente: string;
  agenteColor: string;
  urgencia: DecisionUrgency;
  titulo: string;
  contexto: string;
  riesgoSiNoActua: string;
  impactoEstimado: string;
  options: DecisionOption[];
  pendingQueueCount: number; // cuántas decisiones más hay en cola
}

@Component({
  selector: 'gali-decision-theater',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './gali-decision-theater.component.html',
  styleUrls: ['./gali-decision-theater.component.scss']
})
export class GaliDecisionTheaterComponent {
  decision = input.required<DecisionTheaterData>();
  isLoading = input<boolean>(false);

  onOptionSelected = output<{ signalId: string; optionId: string }>();
  onDecidir        = output<void>(); // "Decidir después"
  onVerCola        = output<void>(); // ver decisiones en cola

  readonly urgencyLabel = computed(() => {
    const map: Record<DecisionUrgency, string> = {
      critical: 'Urgente',
      high:     'Importante',
      medium:   'Recomendado'
    };
    return map[this.decision().urgencia];
  });

  selectOption(optionId: string): void {
    this.onOptionSelected.emit({ signalId: this.decision().signalId, optionId });
  }
}
```

### C3 — Mover `mission__anatomy` (ciclo de negocio) al Momento 3, colapsable por defecto

El bloque de ciclo de negocio (implementado como `mission__anatomy` en el hub actual) pasa al Momento 3. Colapsado por defecto en modo básico, expandido en modo experto.

**Modificar**: `home.component.ts`

Añadir el getter y signal de colapso del Momento 3:

```typescript
// home.component.ts — añadir a la clase existente

import { signal, computed } from '@angular/core';

// Dentro de la clase HomeComponent:

readonly momento3Collapsed = signal<boolean>(true);

readonly complexityLevel = signal<'novice' | 'active' | 'expert'>('active'); // del mock o localStorage

readonly shouldExpandMomento3 = computed(() =>
  this.complexityLevel() === 'expert' || !this.momento3Collapsed()
);

toggleMomento3(): void {
  this.momento3Collapsed.update(v => !v);
}

// Al inicializar, respetar complexityLevel:
ngOnInit(): void {
  const saved = localStorage.getItem('gali_complexity_level') as 'novice' | 'active' | 'expert' | null;
  if (saved) this.complexityLevel.set(saved);
  // En modo experto, Momento 3 arranca expandido
  if (this.complexityLevel() === 'expert') {
    this.momento3Collapsed.set(false);
  }
}
```

### C5 — Regla CSS-level: máximo 1 alerta primaria (ALS-1)

Implementar como directiva Angular que oculta alertas secundarias cuando hay una crítica activa.

**Archivo nuevo**: `src/app/pages/gali-v5/directives/alert-hierarchy.directive.ts`

```typescript
import { Directive, ElementRef, OnInit, OnDestroy, inject } from '@angular/core';
import { GaliSignalService } from '../services/gali-signal.service'; // servicio existente
import { Subscription } from 'rxjs';

/**
 * Oculta el host si existe un gali-alert[data-priority="critical"] activo en la misma página.
 * Uso: <div galiAlertHierarchy data-priority="secondary">...</div>
 */
@Directive({
  selector: '[galiAlertHierarchy]',
  standalone: true
})
export class AlertHierarchyDirective implements OnInit, OnDestroy {
  private el      = inject(ElementRef<HTMLElement>);
  private svc     = inject(GaliSignalService);
  private sub?: Subscription;

  ngOnInit(): void {
    this.sub = this.svc.hasCriticalAlert$.subscribe(hasCritical => {
      const priority = this.el.nativeElement.getAttribute('data-priority');
      if (priority === 'secondary' && hasCritical) {
        this.el.nativeElement.style.display = 'none';
      } else {
        this.el.nativeElement.style.removeProperty('display');
      }
    });
  }

  ngOnDestroy(): void {
    this.sub?.unsubscribe();
  }
}
```

<!-- La alerta crítica siempre visible -->

readonly currentDecision = computed<DecisionTheaterData | null>(() => {
  const pending = this.pendingApprovalSignals();
  if (pending.length === 0) return null;
  // Solo la primera — las demás van en cola
  return this.mapSignalToDecision(pending[0]);

readonly decisionQueueCount = computed(() =>
  Math.max(0, this.pendingApprovalSignals().length - 1)
);

// Texto de la última acción de Gali para el Modo Calma
readonly ultimaAccionGali = computed<string>(() => {
  const actions = this.completedSignals();
  if (actions.length === 0) return 'hace unos momentos';
  const last = actions[0];
  return `hace ${last.tiempoRelativo} · ${last.descripcionCorta}`;
});

private mapSignalToDecision(signal: GaliSignal): DecisionTheaterData {
  return {
    signalId:          signal.id,

    impactoEstimado:   signal.impactoEstimado ?? '',
    options:           signal.opciones ?? [],

  };

| No hacer | Hacer en cambio |
