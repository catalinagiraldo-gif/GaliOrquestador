## 1. Diagnóstico — Lo que está mal hoy

### 1.1 Solo existe el estado "en campaña activa"

`projects.json` tiene proyectos con `estado: "en_escala"` o `"activo"`. No existen proyectos recién lanzados, pausados, cerrados ni en borrador. El usuario no puede distinguir entre fases del ciclo de vida de un proyecto. Un proyecto pausado y uno activo se ven idénticos en la lista, salvo por un texto de estado. La promesa de que Gali "aprende" y "documenta" no tiene representación visual ni de datos.

### 1.2 Tabs vacías con "Ir al módulo →"

**Spec 5 define el contenido base de los tabs**. Spec 13 extiende ese contenido con variantes por estado (`recien_lanzado`, `pausado`, `cerrado`, `borrador`) y añade el componente `proyecto-timeline` y la sección "Acciones de los agentes" que Spec 5 no cubre. La dependencia es: Spec 5 CA-5 en ✅ antes de ejecutar Spec 13 C2.

### 1.3 Ciclo de vida invisible

No existe ninguna representación visual de cuándo se lanzó el proyecto, cuándo escaló, qué pasó en el camino. El usuario entra al detalle y ve datos de hoy — sin historia ni trayectoria. La promesa de "Gali aprende de tu operación" es vacía si no hay un registro cronológico visible.

### 1.4 Calculadora de precio desaparecida

El wizard de nuevo proyecto no tiene cálculo de márgenes. El dropshipper define un presupuesto sin saber si el precio de venta cubre el costo del producto, el flete y la pauta. Gali no puede ser un Director de E-commerce si no ayuda con la aritmética básica del negocio.

### 1.5 "Acciones de los agentes" inexistente por proyecto

El detalle del proyecto muestra "Loops" que son placeholders. No existe una vista de qué hicieron Roax, Vigilante y Kronos específicamente en este proyecto en las últimas 24h. Sin ese feed de actividad, el agente aparece como decorativo y el usuario no puede justificar por qué lo necesita.

---

## 2. Principio de diseño: El proyecto como historia con capítulos

> "El timeline del proyecto convierte la experiencia en una historia con capítulos. El usuario puede ver dónde está en el ciclo de vida, qué pasó antes, y qué viene después. Los proyectos fallidos tienen postmortem narrativo de Gali — no son fracasos, son aprendizajes documentados."
> — Cambios10jun.md §2

Cada proyecto responde: "¿Dónde estoy en esta historia y qué pasó mientras no estaba mirando?"

```typescript
export interface ProyectoTimelineEvent {
  id: string;
  fecha: string;         // ISO date string
  label: string;         // "Lanzado", "Primeros datos", "Escalado", etc.
  descripcion?: string;
  esActual: boolean;     // marca el hito donde está el proyecto
}

export interface ProyectoTimelineData {
  estadoActual: 'borrador' | 'recien_lanzado' | 'activo' | 'en_escala' | 'pausado' | 'cerrado';
  semanaActual?: number; // "Semana 3 de campaña activa"
  eventos: ProyectoTimelineEvent[];
}
```

### `proyecto-timeline.component.ts`

```typescript
import { Component, Input, ChangeDetectionStrategy } from '@angular/core';
import { CommonModule } from '@angular/common';
import { ProyectoTimelineData, ProyectoTimelineEvent } from './proyecto-timeline.model';

@Component({
  selector: 'app-proyecto-timeline',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './proyecto-timeline.component.html',
  styleUrls: ['./proyecto-timeline.component.scss'],
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ProyectoTimelineComponent {
  @Input({ required: true }) timeline!: ProyectoTimelineData;

  get estadoLabel(): string {
    const map: Record<string, string> = {
      borrador:       'Borrador',
      recien_lanzado: 'Recién lanzado',
      activo:         'En campaña',
      en_escala:      'En escala',
      pausado:        'Pausado',
      cerrado:        'Cerrado'
    };
    return map[this.timeline.estadoActual] ?? this.timeline.estadoActual;
  }

  get estadoClass(): string {
    return `estado-pill--${this.timeline.estadoActual}`;
  }

  get contextoLabel(): string {
    if (this.timeline.semanaActual) {
      return `Semana ${this.timeline.semanaActual} de campaña activa`;
    }
    return '';
  }
}
```

## 6. C4 — Calculadora Brújula de precio (wizard de nuevo proyecto)

**Archivo**: `crear-proyecto-modal.component` — agregar un nuevo paso (o sección dentro del paso de presupuesto).

### Señales y computed en `crear-proyecto-modal.component.ts`

```typescript
// Calculadora Brújula
readonly costoProd        = signal<number>(0);
readonly fleteEstimado    = signal<number>(0);
readonly precioSlider     = signal<number>(0);

readonly MARGEN_MINIMO    = 0.15;  // 15%
readonly MARGEN_OBJETIVO  = 0.40;  // 40%

readonly costoTotal = computed(() => this.costoProd() + this.fleteEstimado());

readonly precioMinimo = computed(() => {
  const ct = this.costoTotal();
  if (ct === 0) return 0;
  return Math.ceil(ct / (1 - this.MARGEN_MINIMO));
});

readonly precioRecomendado = computed(() => {
  const ct = this.costoTotal();
  if (ct === 0) return 0;
  return Math.ceil(ct / (1 - this.MARGEN_OBJETIVO));
});

readonly margenProyectado = computed(() => {
  const precio = this.precioSlider();
  const ct = this.costoTotal();
  if (precio === 0 || ct >= precio) return 0;
  return Math.round(((precio - ct) / precio) * 100);
});

readonly margenClass = computed(() => {
  const m = this.margenProyectado();
  if (m >= 40) return 'margen--ok';
  if (m >= 20) return 'margen--warn';
  return 'margen--danger';
});

readonly margenLabel = computed(() => {
  const m = this.margenProyectado();
  if (m >= 40) return `${m}% — Excelente`;
  if (m >= 20) return `${m}% — Mínimo viable`;
  return `${m}% — Bajo riesgo`;
});

// Inicializar slider al calcular precio recomendado
onCostoChange(): void {
  if (this.precioRecomendado() > 0) {
    this.precioSlider.set(this.precioRecomendado());
  }
}

usarPrecioRecomendado(): void {
  this.precioSlider.set(this.precioRecomendado());
  this.precioVenta.set(this.precioRecomendado()); // persiste en el wizard
}
```

```typescript
// ROAS promedio histórico de los proyectos activos
readonly roasHistorico = computed(() => {
  const activos = (PROJECTS as any[]).filter(p =>
    p.estado === 'activo' || p.estado === 'en_escala'
  );
  if (!activos.length) return 2.0;
  const suma = activos.reduce((acc: number, p: any) => acc + (p.roas_real ?? 2.0), 0);
  return suma / activos.length;

readonly pedidosObjetivo = signal<number>(30);

// presupuesto = (pedidosObjetivo * precioVenta) / roasHistorico
readonly presupuestoSugerido = computed(() => {
  const p = this.precioVenta();
  const roas = this.roasHistorico();
  const obj = this.pedidosObjetivo();
  if (!p || !roas || !obj) return 0;
  return Math.ceil((obj * p) / roas / 7); // diario
});
```

## 8. C6 — Modal animado de lanzamiento

Al confirmar el wizard, mostrar una animación de 2s antes de navegar al detalle del proyecto.

### Signal y lógica en `crear-proyecto-modal.component.ts`

```typescript
import { Router } from '@angular/router';

readonly lanzando = signal(false);
readonly lanzandoProgress = signal(0);

private router = inject(Router);

lanzarProyecto(): void {
  this.lanzando.set(true);
  this.lanzandoProgress.set(0);

  // Simular progreso en 2s
  const intervalo = setInterval(() => {
    this.lanzandoProgress.update(v => {
      if (v >= 100) {
        clearInterval(intervalo);
        this.finalizarLanzamiento();
        return 100;
      }
      return v + 5;
    });
  }, 100);
}

private finalizarLanzamiento(): void {
  const nuevoId = this.generarProyectoId();
  setTimeout(() => {
    this.lanzando.set(false);
    this.cerrar.emit();
    this.router.navigate(['/gali-v5/proyectos', nuevoId]);
  }, 300);
}

private generarProyectoId(): string {
  const nombre = this.nombreProyecto().toLowerCase().replace(/\s+/g, '-');
  return `${nombre}-${Date.now()}`;
}
```

## 13. Look & Feel específico

- **Timeline track**: `background: var(--surface-50)` con borde `1px solid var(--surface-200)` — neutral, no llama la atención por sí solo
- **Nodo "actual"**: único elemento naranja del timeline — `background: var(--primary-500)` con halo `box-shadow: 0 0 0 3px rgba(244,154,61,0.2)`
- **Agent actions feed**: sin fondo de sección — las filas flotan sobre el fondo de la página. Solo las amarillas (alertas) tienen `background: var(--amber-50)` + `border-left`
- **Calculadora Brújula**: El precio recomendado es el único valor en naranja — todo lo demás en gris. Esto guía el ojo al número que importa
- **Lanzamiento overlay**: `backdrop-filter: blur(6px)` — el wizard se ve borroso detrás, el modal de lanzamiento toma protagonismo
- **Postmortem "qué no funcionó"**: texto en `var(--red-600)`, no en rojo sólido de fondo — no es alarmante, es documentación
- **Postmortem "lección aprendida"**: en `gali-nota` con fondo naranja suave — la lección es el activo, no el fracaso
- **Toggle de agentes**: toggle naranja al activar, nunca azul ni verde — coherencia con el brand de Gali
