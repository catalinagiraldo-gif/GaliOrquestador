Basado en la conversación transcrita sobre el desarrollo de Gali, estos son los puntos fundamentales que deben guiar el diseño y la identidad de la plataforma para asegurar que se mantenga el enfoque y se simplifique la experiencia de usuario (la "fase uno" consensuada):1. Filosofía de "Menos es Más" (Simplificación)
Reducción de carga cognitiva: El prototipo actual es muy vasto. El objetivo es que la interfaz sea "más ligera", mostrando menos información de golpe para evitar saturar al usuario.1
De "Universo" a "Casita": La estrategia acordada es dejar de lado la idea de un sistema que "lo hace todo" en una sola pantalla compleja. Se debe priorizar una fase uno más pequeña y manejable (el "ranchito en palitos" o "casita en la montaña") que permita validar el valor antes de escalar.1
2. El Centro de la Experiencia: Proyectos y Objetivos
Onboarding basado en objetivos: El modal de inicio debe preguntar al usuario "¿Qué quieres hacer?" (objetivo propio) en lugar de imponer un paso a paso rígido. Esto condiciona la experiencia desde el inicio.1
Visualización de proyectos: El núcleo de la interfaz debe ser la sección de "Proyectos". Aquí, el usuario debe poder:
Ver la salud de cada proyecto (indicadores de "bien/mal").
Identificar rápidamente cuánto falta para cumplir el objetivo del proyecto.
Gestionar agentes, reglas y acciones específicas para cada proyecto.
Conexión Objetivo-Operación: Existe un reto de diseño para conectar los objetivos globales del negocio con la ejecución operativa (el "día a día"). El sistema debe guiar al usuario recomendando proyectos que ayuden a cumplir sus metas de negocio.1
3. Personalización y Modularidad
Dashboards configurables: El usuario no debería ver una interfaz fija. Se deben implementar "pestañitas" para que cada dropshipper (o perfil) arme su propio dashboard según sus necesidades (ej. una pestaña solo para productos, otra para finanzas, otra para pedidos).1
Cambio de modo (Experto/Básico): Mantener la capacidad de alternar entre vistas resumidas y detalladas, evitando el exceso de alertas operativas en la vista básica.1
4. Transparencia en la Autonomía (IA y Agentes)
Configuración de confianza: El diseño debe permitir configurar claramente cuánta autonomía tiene cada agente. El usuario debe poder definir el "umbral de confianza" (si Gali actúa solo o pide permiso).1
Transparencia de Skills y Reglas: El usuario debe poder ver qué agentes están activos, qué skills están usando, qué decisiones han tomado y cuál es su historial reciente. Debe haber una forma clara de editar, activar o desactivar estas capacidades.1
5. El Hub de Conectividad (Marketplace)
Integración centralizada: La interfaz debe reflejar que Dropi no es un ente aislado. Visualmente debe quedar claro cómo se conectan los datos externos (Shopify, Meta, TikTok, etc.) dentro de Dropi.
Marketplace de integraciones: Gali debe mostrarse como el "orquestador" que permite conectar estas APIs y herramientas externas de manera fluida, funcionando como un nodo central de la red de negocio.1
Recomendación de ejecución para la "Fase 1"

Para las próximas iteraciones, el consenso del equipo fue centrarse estrictamente en:
Construir la estructura de Proyectos: Enfocarse en cómo se crea el proyecto, cómo se le da un objetivo y cómo el usuario sigue su progreso.1
Conexión y Contexto: Priorizar la conexión de las 5 herramientas principales (Meta, TikTok, Shopify, Drive, etc.) para que Gali tenga la información necesaria para funcionar, dejando otras integraciones para fases posteriores.
